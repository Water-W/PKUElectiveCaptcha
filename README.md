# PKUElectiveCaptcha (v2)

基于 CNN 的第二版 elective.pku.edu.cn 补选验证码识别，第一版的内容主要内容已经移到 `v1/` 中

## 更新概况

1. 这个 CNN 模型对于已经切分好的字母碎片，大小写敏感的预测准确率达到 **99.88%**
2. 添加了数字 **4** 的标记，现在可以对数字 **4** 进行识别
3. 图像处理流程沿袭了第一版，详细请看 [README](v1/README.md)
4. 训练集/验证集 7:3 划分。训练集为 140514 字母碎片

## 主要问题

1. 根据 [混淆矩阵](#Confusion-Matrix)，模型最主要的问题是不能很好区分 `K/k`, `J/j`，主要的原因是这些字母通过人眼都很难分辨
2. 对于包含 `W`, `M` 等较宽字母的验证码，切割效果不佳，图片的分割算法仍然需要完善


## 开发说明

因为原始数据集比较大 (2G)，在 v2 中我没有上传数据集，并且删掉了 v1 中的数据集，如果您需要下载这个 repo，建议加上 `--depth=1` 以忽略
以前的 commits

```console
$ git clone https://github.com/zhongxinghong/PKUElectiveCaptcha.git --depth=1
```

### 数据集解压

在 `dataset/` 中提供了字母碎片数据集的高压版本，`captcha.bz2` 为未筛选过的数据集，`captcha.user.bz2` 为手工筛选后重新打包的数据集，
解压后的数据格式为，`22*22 bits` 长度的二进制代表一个碎片的图像，`0` 表示一个 `(0,0,0)` 像素，`1` 表示一个 `(255,255,255)` 像素，
紧跟着的 `1 bytes` 代表该图像碎片标记的 ascii 码。

您可以通过 `filter_dataset.py` 中的 `unpack_dataset` 函数解压这种数据集（需要手动指定其中的数据集文件路径，默认指向 `captcha.bz2`，
建议您将其更改为指向 `captcha.user.bz2`），通过 `pack_dataset` 将 `unpack_dataset` 得到的 `decoded/` 数据集重新打包成 `captcha.user.bz2`

您可以对 `decoded/` 中的图片进行手动筛选，然后重新打包成新的 `captcha.user.bz2`


### 数据集下载和构建

如果您需要从头构建数据集，可以借鉴 `fetch_captcha.py` 自行下载原始数据集，然后借鉴 `build_dataset_by_svm.py` 对原始数据集进行切割
和初步标记（建议用已经得到的 CNN 模型来代替原始的 SVM 模型对原始数据集进行初步标记，这样得到的结果会准确很多），并打包成 `captcha.bz2`，
最后再利用 `filter_dataset.py` 中的函数打包/解压所得的数据集。此时您可以对解压出来的 `decoded/` 数据集进行手工标记和筛选


### 模型训练

在 `cnn.py` 中提供了模型的训练/验证相关函数，会使用 `captcha.user.bz2 / captcha.bz2` 数据集训练模型（优先使用 `captcha.user.bz2`），
得到的模型保存为 `model/cnn.pt.gz`
